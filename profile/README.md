# Polychrome

> This is my personal studio — [@inkxel](https://github.com/inkxel) is where the rest of me lives.

I've spent fifteen years moving between entertainment, tech, and agency work — type design, creative direction, systems building — and at some point stopped treating those as different jobs. Polychrome is where that shows up on its own, outside of client work.

Two things happen here. I take on creative, strategy, and technology work directly — the same blend, just without an agency's org chart sitting on top of it. And I build the tools and open standards that work runs on, in the open, so anyone else building similar things doesn't have to start from zero.

*Precise by default. Chromatic on discovery.*

Most of what's here is quiet on purpose — black, white, exact. The name is the joke: many colors, almost none of them visible until you actually look.

## What's here

- [foundry](https://github.com/polychrome-studio/foundry) — a portable, markdown-native work harness. The room an agent runs inside.
- [agent-kit](https://github.com/polychrome-studio/agent-kit) — an agent is a directory. The open format for defining one.
- [dotKnowledge](https://github.com/polychrome-studio/dotKnowledge) — an open, portable knowledge-bundle format, OKF-conformant.
- [knowledge-kit](https://github.com/polychrome-studio/knowledge-kit) — the template for a `.knowledge` capsule: self, client/brand, or project.
- [brand-kit](https://github.com/polychrome-studio/brand-kit) — the method for codifying a brand's voice, personas, motion, and visual identity.
- [motion.md](https://github.com/polychrome-studio/motion.md) — the motion sibling of Google's `design.md`.
- [skills-kit](https://github.com/polychrome-studio/skills-kit) — how a skill gets addressed from an agent or a capsule.

More of this is still private, mid-sort. It opens up as it's ready.
